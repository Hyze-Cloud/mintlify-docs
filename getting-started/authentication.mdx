Use an API key when you call Hyze Cloud from CI, servers, scripts, or internal tools.

## Goal

Send an authenticated request to the Hyze Cloud API.

## When to use

Use API keys for server-side integrations.

Do not expose API keys in browser code.

## Example

```bash
curl https://api.hyze.cloud/api/apps/ \
  -H "Authorization: Bearer hyze_your_key"
```

## How it works

Every private endpoint reads the API key from the `Authorization` header.

```http
Authorization: Bearer hyze_your_key
```

<Tip>
  Store the key in an environment variable like `HYZE_API_KEY`.
</Tip>

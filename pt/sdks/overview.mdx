As SDKs oficiais encapsulam a REST API da Hyze Cloud com helpers tipados, erros consistentes e menos boilerplate.

A API continua sendo a fonte da verdade. As SDKs cobrem as superfícies principais: apps, databases, API keys, invoices, GitHub e plans.

## Pacotes oficiais

| Pacote | Runtime | Status | Docs |
| --- | --- | --- | --- |
| [`@hyzecloud/sdk`](/pt/sdks/typescript) | Node.js 18+, Bun | **Disponível** | [Guia](/pt/sdks/typescript) |
| Python | Python 3.10+ | Em breve | — |

## Instalação (Node / Bun)

```bash
npm install @hyzecloud/sdk
# ou
bun add @hyzecloud/sdk
```

```ts
import { HyzeCloud } from "@hyzecloud/sdk";

const hyze = new HyzeCloud({
  apiKey: process.env.HYZE_API_KEY, // hyze_...
});

const { apps } = await hyze.apps.list();
```

## Quando usar uma SDK

- Quer helpers tipados de request/response
- Prefere `hyze.apps.restart(id)` em vez de `fetch` cru
- Precisa de erros consistentes (`HyzeError`, rate limits)

Use a [API Reference](/api-reference/introduction) (ou curl) quando quiser controle total ou uma linguagem sem SDK ainda.

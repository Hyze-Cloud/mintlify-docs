Este guia leva você de uma conta nova no Dashboard até uma requisição autenticada funcionando.

## Objetivo

Verificar se sua chave de API funciona.

## Quando usar

Use isto antes de conectar a Hyze Cloud em CI/CD, scripts ou seu backend.

## Pré-requisitos

Antes de começar, você precisa ter:

- Uma conta na Hyze Cloud ([cadastre-se](https://app.hyze.cloud))
- Uma chave de API (começa com `hyze_`)

<Callout type="info">
  A URL base da API é `https://api.hyze.cloud/api`. Mantenha esse prefixo em todas as requisições.
</Callout>

## Obter sua chave de API

<Steps>
  <Step title="Acesse o Dashboard">
    Vá para [app.hyze.cloud](https://app.hyze.cloud) e faça login.
  </Step>
  <Step title="Navegue até API Keys">
    Abra **Settings → API Keys** e clique em **Create Key**.
  </Step>
  <Step title="Copie sua chave">
    Sua chave começa com `hyze_`. Copie-a — ela não será mostrada novamente.
  </Step>
</Steps>

## Faça sua primeira requisição

<Steps>
  <Step title="Liste seus apps">
    ```bash
    curl https://api.hyze.cloud/api/apps/ \
      -H "Authorization: Bearer hyze_sua_chave"
    ```

    Você deve receber uma resposta JSON com os apps do seu workspace.
  </Step>
</Steps>

## O que acontece em seguida

A API retorna seus apps atuais.

Se esta requisição funcionar, a autenticação está configurada corretamente.

## Verifique seu ambiente

Use este trecho para confirmar que sua chave está disponível antes de usá-la em CI/CD.

```bash
export HYZE_API_KEY="hyze_sua_chave"

curl https://api.hyze.cloud/api/apps/ \
  -H "Authorization: Bearer $HYZE_API_KEY"
```

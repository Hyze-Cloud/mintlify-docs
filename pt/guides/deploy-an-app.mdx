## Deploy a partir de um ZIP

<Steps>
  <Step title="Empacote seu app">
    Crie um ZIP com os arquivos que a Hyze Cloud deve executar. Inclua o entrypoint do seu app e o manifesto de dependências.
  </Step>
  <Step title="Envie a requisição de deploy">
    ```bash
    curl -X POST https://api.hyzecloud.com/api/apps/deploy \
      -H "Authorization: Bearer hyze_sua_chave" \
      -F "file=@app.zip" \
      -F "name=my-api" \
      -F "runtime=node" \
      -F "memoryMB=512" \
      -F "exposePort=3000"
    ```

    `startupCommand` é opcional. Omita (ou use `auto`) para a Hyze detectar o comando certo. Override só se precisar: `-F "startupCommand=node server.js"`.
  </Step>
  <Step title="Veja os logs">
    ```bash
    curl "https://api.hyzecloud.com/api/apps/app_001/logs?tail=200&timestamps=true" \
      -H "Authorization: Bearer hyze_sua_chave"
    ```
  </Step>
</Steps>

## Deploy pelo GitHub

Use [`POST /api/apps/deploy-from-repo`](/api-reference/apps#deploy-from-github-repo) após conectar o GitHub no Dashboard.

```json
{
  "name": "my-api",
  "runtime": "node",
  "memoryMB": 512,
  "exposePort": 3000,
  "autoDeploy": true,
  "repository": {
    "id": "repo_001",
    "owner": "hyzecloud",
    "name": "my-api",
    "branch": "main"
  }
}
```

<Tip>
  Defina `autoDeploy` como `true` quando quiser que a Hyze Cloud faça redeploy após pushes na branch selecionada.
  Omita `startupCommand` (ou use `"auto"`) a menos que precise de um comando custom.
</Tip>

## Checklist de runtimes

| Runtime | Auto detect | Observações |
| --- | --- | --- |
| `node` | Sim (API, site estático, Vite, Next…) | Exponha a mesma porta em que seu servidor escuta. |
| `bun` | Sim | Mantenha o lockfile no artefato quando possível. |
| `python` | Sim | Inclua seu arquivo de dependências no ZIP. |

Exemplos de override opcional: `node server.js`, `bun run start`, `python main.py`.

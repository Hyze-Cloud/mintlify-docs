## Preciso de uma chave de API?

Sim. Endpoints privados exigem `Authorization: Bearer hyze_sua_chave`.

## Qual é a URL base da API?

Use `https://api.hyzecloud.com/api`.

## Quais runtimes são suportados?

Apps suportam Node.js, Bun e Python.

## Quais bancos de dados são suportados?

A Hyze Cloud suporta PostgreSQL, MySQL, MongoDB e Redis.

## Devo usar a API ou o Dashboard?

Use o Dashboard para operações manuais.

Use a API para CI/CD, ferramentas internas e automação.

## Onde os segredos devem ficar?

Armazene segredos em variáveis de ambiente do app ou em seu gerenciador de segredos externo.

Nunca os commite no controle de versão.

## O que significa HTTP 429 no meu site?

A Hyze aplica **rate limit no proxy público**. Muitas requisições do mesmo IP (ou tráfego total alto no subdomínio) retornam `429` com `Retry-After`.

Isso **não** significa que o app foi pausado — o processo continua rodando. Veja [Rate limits](/pt/concepts/rate-limits).

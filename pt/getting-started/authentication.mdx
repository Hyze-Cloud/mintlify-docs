title: Autenticação

---

Endpoints privados exigem uma API key no header `Authorization`.

```http
Authorization: Bearer hyze_sua_chave
```

## Criar uma key

1. Abra [app.hyze.cloud](https://app.hyze.cloud)
2. **Settings → API Keys → Create Key**
3. Copie o secret uma vez (`hyze_...`)

Keys ficam ligadas a um **workspace**. As requisições usam os recursos desse workspace.

## Exemplo

```bash
curl https://api.hyzecloud.com/api/apps/ \
  -H "Authorization: Bearer hyze_sua_chave"
```

Com a SDK, passe a key uma vez:

```ts
import { HyzeCloud } from "@hyzecloud/sdk";

const hyze = new HyzeCloud({
  apiKey: process.env.HYZE_API_KEY,
});
```

## Boas práticas

- Guarde keys em variáveis de ambiente (`HYZE_API_KEY`), nunca no código ou no front-end
- Use uma key por ambiente ou automação (CI, local, tooling de produção)
- Revogue keys que não usa mais

## Deploy from a ZIP

<Steps>
  <Step title="Package your app">
    Create a ZIP with the files Hyze Cloud should run. Include your app entrypoint and dependency manifest.
  </Step>
  <Step title="Send the deploy request">
    ```bash
    curl -X POST https://api.hyze.cloud/api/apps/deploy \
      -H "Authorization: Bearer hyze_your_key" \
      -F "file=@app.zip" \
      -F "name=my-api" \
      -F "runtime=node" \
      -F "memoryMB=512" \
      -F "startupCommand=node server.js" \
      -F "exposePort=3000"
    ```
  </Step>
  <Step title="Watch logs">
    ```bash
    curl "https://api.hyze.cloud/api/apps/app_001/logs?tail=200&timestamps=true" \
      -H "Authorization: Bearer hyze_your_key"
    ```
  </Step>
</Steps>

## Deploy from GitHub

Use [`POST /api/apps/deploy-from-repo`](/api-reference/apps#deploy-from-github-repo) after connecting GitHub in the Dashboard.

```json
{
  "name": "my-api",
  "runtime": "node",
  "memoryMB": 512,
  "startupCommand": "npm start",
  "exposePort": 3000,
  "autoDeploy": true,
  "repository": {
    "id": "repo_001",
    "owner": "hyzecloud",
    "name": "my-api",
    "branch": "main"
  }
}
```

<Tip>
  Set `autoDeploy` to `true` when you want Hyze Cloud to redeploy after pushes to the selected branch.
</Tip>

## Runtime checklist

| Runtime | Typical startup command | Notes |
| --- | --- | --- |
| `node` | `node server.js` or `npm start` | Expose the same port your server listens on. |
| `bun` | `bun run start` | Keep the lockfile in your artifact when possible. |
| `python` | `python main.py` | Include your dependency file in the ZIP. |
